• 1. <abbr>

    行内元素，表示简称或缩写，如果使用title属性，在不同浏览器上表现不一致

  • 2. <address>

    块级元素，倾斜，联系信息，位于body内（文档的）和位于article内（文章的）不同

  • 3. <i><b><small>字体样式元素

    行内元素，i倾斜，b加粗，small变小(不同浏览器不一样)

  • 4. <bdo>

    行内元素，修改默认文本方向

  • 5. <blockquote>

    块级元素，块引用，一般包含p标签，cite属性无浏览器支持（搜索引擎可能用到）

  • 6. <q>

    行内元素，短标记，cite属性无浏览器支持（搜索引擎可能用到）

  • 7. <em><strong><dfn><code><samp><var><cite>短语元素

    行内元素，em斜体、强调，strong加粗、强调、重要文本使用，dfn斜体、定义项目，code等宽字体、显示计算机源码，samp从上下文中提取文本，var斜体、定义变量，cite斜体、作品标题

  • 8. <ins><del>

    行内元素，cite和datetime属性无浏览器支持，ins定义新插入部分的文本，del定义已删除部分的文本

  • 9. <pre>

    块级元素，预格式化文本，等宽字体，保留空格和换行符，内容可包含文本或行内元素，可用链接、图片、水平分割线、符号实体

  • 10. <s>

    行内元素，对不正确、不准确或没用的文本进行标识，HTMl5重定义

  • 11. <u>

    行内元素，HTMl5重定义，不建议使用，下划线

  • 12. <sub><sup>

    行内元素，sub下标注，sup上标注

  • 13. <bdi>

    HTML5新增，IE9+支持，行内元素，文本脱离父元素文本设置方向

  • 14. <bdi>

    HTML5新增，IE9+支持，行内元素，带有标记的文本，yellow背景

  • 15. <meter>

    HTML5新增，IE不支持，行内元素，度量衡

  • 16. <progress>

    HTML5新增，IE10+支持，行内元素，任务进度（与JavaScript一起使用）

  • 17. <time>

    HTML5新增，IE9+支持，行内元素，定义时间（24小时制）或日期

  • 18. <wbr>

    HTML5新增，IE不支持，行内元素，规定文本何处添加换行符

  • 19. <ruby><rt><rp>

    HTML5新增，IE9+支持，行内元素，ruby注释，与rt一起使用，rp可选，定义不支持ruby时的显示

abbr是一个简称/缩写标签。


address标签用来设置作者/所有者的联系信息;
在body内，表示该文档的作者/所有者的联系信息；
在article内表示该文章的作者/所有者的联系信息；
通常被包含在footer元素内
通常显示为斜体
不应该用它描述邮政地址，除非这些信息是联系信息的组成部分
如DuDujie


b标签用来显示一个加粗文本，它应该作为最后的选择，只有没有合适标记时使用；
标题用h1~h6，被强调的文本用em，重要的文本用strong，被标记的或高亮显示的文本用mark；


bdi(新)标签用来隔离，设置一段文本，使其脱离父元素的文本方向设置；
Edge不支持；
CSS属性unicode-bidi可以实现此功能，但是语义信息只能通过bdi标签实现；
在发布用户评论或其它无法完全控制的内容时，很有用；
dir属性设置bdi标签的文本方向（似乎无用）
默认auto
值ltr
值trl

blockquote标签定义摘自另一个源的块引用；
q标签定义一个短的引用；
它们两个本质上一样，不同之处在于它们的显示和应用：q标签用于简短的行内引用；若需要从周围内容分离出来较长的部分（通常显示为缩进的块），使用blockquote标签；
浏览器通常对blockquote标签进行缩进；有时会使用斜体；
浏览器通常在q标签的周围加上引号；
有属性cite表示引用的来源，但是当前主流浏览器都不支持此属性；搜索引擎可能会用到该属性；可通过屏幕阅读器使用；

cite标签定义作品（如书籍、歌曲、电影、电视节目、绘画、雕像等）的标题；
按照惯例，引用的文本将以斜体显示；
它有一个隐藏功能：它可以从文档中自动摘录参考书目；
《JavaScript权威指南》是一部非常经典的JS入门书籍；


code标签定义计算机代码文本；
var s = "Hello World!";
.code { font-size: 24px; line-hiehgt: 1.5; }


del标签定义文档中已删除的文本；
ins标签定义已经被插入文档中的文本；
它们通常一起使用，描述文档中的更新和修正；
浏览器通常在已删除文本上添加一条删除线，在新插入文本下添加一条下划线；
有属性cite（值url，解释文档被插入的原因）和datetime（值YYYY-MM-DDThh:mm:ssTZD，规定文本被插入的日期和时间）；这两个属性没有主流浏览器支持，但可使用于屏幕阅读器；

这是插入删除的标签演示。



dfn标签定义一个定义项目（标记对特殊术语或短语的定义）；
浏览器通常以斜体显示；它可能有助于创建文档的索引或术语表；尽量少用此标签；
在技术文档中，第一次新的术语，使用此标签，后面对这个术语就不需要进行任何标记了；
定义项目


em用来呈现被强调的文本；
通常显示为斜体；不要滥用；
除强调外，当引入新的术语或在引用特定类型的术语或概念时作为固定样式的时候，可以考虑使用em标签；
em标签显示的内容


i标签显示斜体文本样式；
它告诉浏览器将包含其中的文本一斜体字（italic）或者倾斜（oblique）字体显示；如果这种斜体字对该浏览器不可用的话，可以使用高亮、反白或加下划线等样式；
